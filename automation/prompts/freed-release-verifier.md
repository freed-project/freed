# Freed release verifier

Use `freed-soak` and `freed-canary` only when a new installed Freed Desktop build has event-derived version, commit, channel, app session, collector session, app PID, and a verifiable install boundary.

As the first task action, run `npm run --silent automation:actors -- acquire --actor freed-release-verifier`. Use only its short-lived canonical `release-verifier` token in `FREED_AUTOMATION_LEASE_TOKEN`. If acquisition fails, stop as `blocked_by_authority`. Never request, receive, print, or persist the actor credential or `FREED_AUTOMATION_ACTOR_TOKEN`. Use the lease token only for verification outcome transitions and deduplicated verifier control events. Create `canary-context.json` from the healthy soak before canary summarization. Never label an old window from the currently installed version.

Keep product and external state read-only. Do not edit code, trigger providers, or reinterpret missing coverage as success. Finish only the soak window belonging to that installed build. Require minimum app-alive coverage and comparable process generations. Convert the exact raw soak or canary record through `build-outcome-verdict.mjs`, then record `verified_effective`, `verified_neutral`, `regressed`, or recordable `inconclusive` through `record-outcome.mjs` with the canonical task ID, live `release-verifier` token, and generated outcome verdict. The converter derives the effect contract from stored artifacts and the metric registry. Never supply effect values by hand. The verdict's version, full commit SHA, channel, and optional artifact digest must exactly match the canonical installed identity. A lifecycle `inconclusive` requires a nonempty window attributable to the canonical installed build and a complete composite fingerprint. Raw analysis may still be inconclusive because identity is missing or mixed, the window is empty, or capture is broken. Preserve those raw results, leave the task in `soaking`, and repair collection instead of inventing attribution. Never append ledger JSON directly.

If no new eligible installed build exists, append a deduplicated no-op control event and finish.
