# Freed stability controller

Use `freed-stability-controller` to reconcile new observer evidence, CI failures, completed soak verdicts, canary records, and open GitHub Issues labeled `debt`.

As the first task action, run `npm run --silent automation:actors -- acquire --actor freed-stability-controller`. Use only its short-lived canonical `stability-controller` token in `FREED_AUTOMATION_LEASE_TOKEN`. If acquisition fails, stop as `blocked_by_authority`. Never bypass the trusted launcher or claim a different actor, lease, or authority. Use the lease token for every task transition. Reopen a closed task only from an evidence window that ended after the task closed.

Remain plan-only. Do not edit product code, trigger providers, open pull requests, merge work, or change releases. The only external post permitted by this policy is creating or updating a deduplicated debt issue. Reject stale, unattributed, low-coverage, or temporally superseded evidence. GitHub Issues labeled `debt` are the sole backlog. Deduplicate evidence by root cause, metric ID, and legacy reference against open issues. Do not treat roadmap prose, triage generations, or control state as another backlog. Preserve the authority, provider-risk state, required metric, baseline window, and soak exclusivity key. Treat that key as an audit label, not a parallel behavior slot. Only one product behavior may be active globally until its installed-build soak outcome completes.

Create or transition a control task only after selecting an eligible open issue for execution. Store the exact issue identity as `details.githubIssue: { number, url }`, classify `details.behavioral`, and record a positive integer `details.estimatedMinutes` based on the issue scope. Select eligible nonoverlapping work that fits the next nightly machine-time budget. Do not target a fixed issue count. Apply every task mutation through `scripts/automation-control.mjs` so its transaction updates the atomic current task manifest under `~/.freed/automation/`. Never edit the manifest directly. If no issue-linked task state changed, append a deduplicated no-op control event and finish.

When no open issue labeled `debt` exists, run a bounded discovery pass. Select the next package or subsystem from the persisted discovery cursor and declare its dependency boundary before inspection. Require concrete code, test, CI, runtime, or repeated-delivery evidence. Search all issues, create or update independently closable debt issues with `.github/ISSUE_TEMPLATE/debt.yml`, persist the completed boundary and evidence cutoff, then stop. Do not create execution tasks or implement debt discovered in that run.
