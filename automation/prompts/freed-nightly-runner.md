# Freed nightly executor

Run the approved Freed maintenance loop from current `origin/dev`. Use concrete repository, CI, runtime, and task evidence. Load the atomic current task manifest. Do not infer work from roadmap prose or stale automation memory.

As the first task action, run `npm run --silent automation:actors -- acquire --actor freed-nightly-runner`. Use only its short-lived canonical `nightly-writer` token in `FREED_AUTOMATION_LEASE_TOKEN`. If acquisition fails, stop as `blocked_by_authority`. Never bypass the trusted launcher or claim a different actor, lease, or authority. Use the writer lease token for every control-state mutation. Execute only candidates backed by a runnable canonical task whose authority permits action. Safe scaffolding tasks may be grouped when they do not change runtime behavior. Execute at most one product behavior change globally. A different soak exclusivity key does not create another slot. Wait until the active change has an authenticated installed-build soak outcome before starting the next behavior. Never execute provider-visible work unattended. Prepare Gate 1 behavior approval before implementation and Gate 2 exact-diff authorization after commit. Each gate requires its own scoped approval record. Keep provider-visible pull requests draft until CODEOWNER review. Otherwise stop at the appropriate approval packet.

Use the matching Freed build skill, focused tests, and `validate:feature`. A merged result is not an effective result until its installed-build metric moves. Record each merge or install transition through `record-outcome.mjs` with the canonical task ID, live `nightly-writer` token, and exact merge or install evidence digest. An install record must include version, full commit SHA, channel, and the artifact digest when available. Verification belongs to the release verifier and must reference its exact JSON verdict file. Never append ledger JSON directly. Never reward a task merely because similar work shipped.

For external posts, begin the body with `(AI Generated).` and keep AI references out of titles, branches, and labels. Public roadmap presentation belongs in a separate `www` lane using `docs/roadmap-status.json`, never in a dev worktree. A response timeout for routine local interaction never grants provider approval or authority for authentication, external posting, deployment, destructive state changes, or a new behavior.

If nothing safe is actionable, append a deduplicated no-op control event with the inspected cutoff, release the writer lease, and archive the automation task.
